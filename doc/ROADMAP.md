# 実装ロードマップ（アジャイル方式）

## 📌 方針
- **MVP（Minimum Viable Product）ファースト**: 最小構成で素早く公開
- **段階的改善**: 各Sprintで完成させ、ユーザーフィードバックを反映
- **1Sprint = 1週間（3-5営業日）を目安に実装**

---

## 🚀 Sprint 1: MVP版 - 基本のダメージ計算機（1週間）

### 目標
最小構成で「動作するダメージ計算機」を完成させ、公開状態を維持

### タスク

| # | タスク | 見積 | ステータス |
|---|--------|------|-----------|
| 1-1 | ポケモン10体データ作成（基本統計情報） | 1h | - |
| 1-2 | 複数技選択対応（技データを整備） | 1h | - |
| 1-3 | レベル選択機能（50固定 → カスタマイズ） | 0.5h | - |
| 1-4 | ダメージ計算式の精度向上テスト | 1h | - |
| 1-5 | 公開サイトへのデプロイ | 0.5h | - |

### 完成基準 ✓
- [ ] ポケモン10体が正常に選択できる
- [ ] 技を複数から選択可能
- [ ] レベル入力により異なるダメージが計算される
- [ ] 公開サイトで正常に動作
- [ ] コンソールエラーなし

### 成果物
- `pokemon.json` v1.0（10体）
- `moves.json` v1.0（10～15種類）
- 更新版 `index.html` / `main.js`

**見積合計**: 4h（2営業日）

## 🔄 継続的改善フェーズ

各Sprintの完成後、以下で評価＆フィードバック反映:

### 確認項目
- [ ] Google Analyticsでのアクセス数・行動確認
- [ ] ユーザーからの問い合わせ・フィードバック
- [ ] ページ読み込み速度の計測
- [ ] SEO検索順位の確認

### 次フェーズの優先順位（フィードバック次第で変更）

1. **複数技比較機能** - 「複数の技をまとめて比較したい」要望が多い場合
2. **相性チェッカー** - 「このポケモンに有利なポケモンは？」要望が多い場合
3. **ポケモン全国図鑑対応** - アクセス数が安定した場合
4. **難易度計算機など追加機能** - ユーザー要望に応じて

## 📅 実装スケジュール案

```
Week 1 (Now)
├─ Sprint 1: MVP版
└─ 公開維持

Week 2
├─ Sprint 2: タイプ相性
└─ アナリティクス確認

Week 3
├─ Sprint 3: 検索＆拡充
└─ ユーザーフィードバック収集

Week 4
├─ Sprint 4: SEO対策
└─ AdSense申請

Week 5+
└─ 継続的改善（フィードバック反映）
```

---

## 💡 アジャイルのメリット

✅ 最小限の機能で素早く公開 → 実際のユーザーフィードバック獲得  
✅ 各Sprint終了時に動作する成果物 → リスク低減  
✅ 途中で優先順位を変更可能 → 市場変化に対応  
✅ 定期的なリリース → SEO効果も段階的に得られる
